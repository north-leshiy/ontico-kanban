## ADDED Requirements

### Requirement: Колонки по decisions
Канбан-доска SHALL отображать колонки на основе массива `decisions` из API метаданных. Каждая колонка соответствует одному decision (Подан, Созвон с куратором, Доработка, К обсуждению, К принятию, Приняли, К отклонению, Отклонили). Порядок колонок SHALL соответствовать порядку в API.

#### Scenario: Отображение колонок
- **WHEN** метаданные загружены
- **THEN** отображаются 8 колонок с заголовками из `decision.name`, каждая со счётчиком карточек

#### Scenario: Карточки в правильных колонках
- **WHEN** заявки загружены
- **THEN** каждая карточка отображается в колонке, соответствующей `lecture.decision.variant`

### Requirement: Drag & Drop смена decision
Пользователь SHALL иметь возможность перетаскивать карточки между колонками с помощью @dnd-kit. Перетаскивание SHALL вызывать API для смены decision.

#### Scenario: Успешное перетаскивание
- **WHEN** пользователь перетаскивает карточку из одной колонки в другую и отпускает
- **THEN** карточка оптимистично перемещается в новую колонку и запускается API-запрос смены decision

#### Scenario: Визуальный feedback при перетаскивании
- **WHEN** пользователь держит карточку над другой колонкой
- **THEN** целевая колонка выделяется визуально (highlight), карточка показывается полупрозрачной

#### Scenario: Перетаскивание внутри одной колонки
- **WHEN** пользователь перетаскивает карточку в ту же колонку
- **THEN** ничего не происходит, API не вызывается

### Requirement: Фильтрация по секции TechLead
Доска SHALL показывать ТОЛЬКО заявки с `section.value === 10634778` (TechLead). Заявки других секций SHALL быть исключены из отображения.

#### Scenario: Фильтрация применена
- **WHEN** заявки загружены (включая заявки других секций)
- **THEN** на доске отображаются только заявки с section.name === "TechLead"

### Requirement: Горизонтальный скролл
Доска SHALL поддерживать горизонтальный скролл если колонки не помещаются на экране. Высота каждой колонки SHALL использовать весь доступный viewport.

#### Scenario: Скролл при большом количестве колонок
- **WHEN** канбан-доска открыта на экране стандартного ноутбука (1366px)
- **THEN** все колонки доступны через горизонтальный скролл, вертикальный скролл внутри каждой колонки работает независимо

### Requirement: Состояние загрузки и ошибок
Доска SHALL отображать состояния загрузки и ошибок на уровне всего экрана.

#### Scenario: Начальная загрузка
- **WHEN** страница только открыта
- **THEN** отображается полноэкранный индикатор загрузки пока данные не получены

#### Scenario: Ошибка загрузки
- **WHEN** API вернул ошибку или пользователь не авторизован
- **THEN** отображается понятное сообщение об ошибке с инструкцией (авторизоваться на conf.ontico.ru) и кнопкой повтора
