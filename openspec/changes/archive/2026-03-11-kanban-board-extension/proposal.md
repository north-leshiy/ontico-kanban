## Why

Члены программного комитета (ПК) конференции ontico.ru работают с заявками докладчиков через табличный интерфейс conf.ontico.ru/lectures/review, который неудобен для потокового разбора заявок. Разработка нового интерфейса силами ontico экономически нецелесообразна, поэтому нужно решение на стороне пользователя — браузерное расширение с канбан-доской.

## What Changes

- Новое браузерное расширение Chrome (Manifest V3) для conf.ontico.ru
- Расширение добавляет отдельную страницу с канбан-доской заявок
- Колонки канбана = решения по заявке (`decision`): Подан → Созвон с куратором → Доработка → К обсуждению → К принятию → Приняли / К отклонению → Отклонили
- Перетаскивание карточки между колонками вызывает API для смены `decision`
- Показываются только заявки из секции TechLead (id: 10634778, конференция Saint TeamLead Conf 2026)
- Background service worker проксирует запросы к API, используя куки браузера пользователя (обход CORS, каждый член ПК работает со своим аккаунтом)

## Capabilities

### New Capabilities

- `extension-shell`: Структура браузерного расширения (manifest.json MV3, background service worker, точка входа страницы)
- `api-client`: Клиент для API conf.ontico.ru — загрузка метаданных (`get_info.json`) и заявок (`moderate2.json`, все страницы), смена decision заявки
- `kanban-board`: Канбан-доска на React + @dnd-kit с колонками по decisions, фильтрацией по секции TechLead
- `lecture-card`: Карточка заявки: название, имя спикера, аватары кураторов из ПК

## Impact

- Новый проект: браузерное расширение Chrome (не модификация существующего кода ontico)
- Зависимости: React 19, Vite, TypeScript, @dnd-kit/core, @dnd-kit/sortable, Tailwind CSS
- API: `GET /api/lectures/get_info.json`, `POST /api/lectures/moderate2.json`, предположительно `POST /api/lectures/<id>/set_decision` (или аналог) для смены статуса
- Куки пользователя используются автоматически service worker'ом — дополнительная авторизация не нужна
