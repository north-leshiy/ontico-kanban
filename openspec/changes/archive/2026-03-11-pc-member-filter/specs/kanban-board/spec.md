## ADDED Requirements

### Requirement: Фильтрация карточек по выбранным кураторам
Когда активен фильтр по кураторам, доска SHALL отображать только карточки, у которых в `curator.value` есть хотя бы один из выбранных ID кураторов. Карточки без совпадений SHALL быть скрыты из всех колонок. Счётчик карточек в заголовке колонки SHALL отражать количество видимых карточек.

#### Scenario: Фильтр по одному куратору
- **WHEN** выбран один куратор в панели фильтра
- **THEN** в каждой колонке показываются только карточки с этим куратором в `curator.value`; колонки с нулевым результатом остаются видимы (пустые)

#### Scenario: Фильтр по нескольким кураторам
- **WHEN** выбрано несколько кураторов
- **THEN** карточка видна если хотя бы один из `curator.value` совпадает с выбранными (OR-логика)

#### Scenario: Фильтр не активен
- **WHEN** ни один куратор не выбран
- **THEN** доска показывает все карточки (поведение без изменений)

#### Scenario: Drag & Drop при активном фильтре
- **WHEN** пользователь перетаскивает карточку при активном фильтре
- **THEN** оптимистичный апдейт decision выполняется корректно; после смены decision карточка остаётся скрытой если не соответствует фильтру
